---
title: Security model
description: Understand credentials, actions, network boundaries, and responsible disclosure.
---

Dashbored is designed for a trusted private network. It is not a public-facing administration portal.

## Credentials at rest

Upstream credentials are encrypted with AES-256-GCM before storage in PostgreSQL. The nonce, ciphertext, authentication tag, and key version are stored separately. The master key lives only in `CREDENTIAL_MASTER_KEY` outside the database.

- The browser never receives saved service credentials.
- Structured logs must redact secrets, cookies, tokens, and query parameters.
- Losing the master key makes existing credentials unrecoverable.
- Back up the key securely and separately from the database.

## Network boundary

Dashbored only makes outbound adapter requests to configured private or Tailscale destinations. It blocks loopback, link-local, public, redirect-pivot, and URL-embedded credential targets. This reduces the risk of a compromised configuration turning Dashbored into an SSRF relay.

PostgreSQL stays on an internal Compose network. Do not add a database host port for convenience.

## Authentication and actions

Operational mutations require an authenticated owner session, origin validation, explicit confirmation, duplicate-submission protection, rate limiting, and an audit event. Dashbored never mounts `/var/run/docker.sock`; container actions go through a scoped Portainer token.

Use a read-only Portainer token if you do not need actions. Keep the owner password and the Portainer token distinct.

## Deployment baseline

- Keep the dashboard on a LAN, Tailscale, or another private overlay.
- Use HTTPS and a stable hostname before using passkeys.
- Restrict the host port with a firewall or private reverse proxy.
- Run the Docker daemon and the Dashbored project under a trusted administrator account.
- Keep `.env`, database dumps, and backups out of Git, screenshots, and public issue reports.
- Apply updates after reviewing release notes and taking a backup.

## What Dashbored does not do

- It does not expose a generic upstream proxy to the browser.
- It does not mount or manage the raw Docker socket.
- It does not disable TLS verification globally for self-signed services.
- It does not make public exposure safe by itself.

## Report a vulnerability

Do not file public issues containing exploit details, private URLs, API tokens, or credentials. Use the repository's private vulnerability-reporting feature as described in `SECURITY.md`. If that channel is unavailable, open a minimal public issue requesting a private contact route.
