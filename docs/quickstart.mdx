## Requirements

- Docker Engine with Compose v2
- Bun 1.3 or newer
- A private hostname or LAN address for Dashbored
- A PostgreSQL volume with enough space for compact health history
- A scoped Portainer access token for discovery

## Configure

Create a private `.env` from `.env.example`, generate unique values for every
secret, and set the correct external address before starting Compose. Never
reuse a server login password as an application secret.

```bash
bun run serve
```

Open the configured dashboard URL and sign in with the bootstrap owner account. A fresh Overview starts empty on purpose: choose **Connect Portainer**, then the worker will populate Launcher after the first successful inventory sync.

## First-run checklist

1. Run `docker compose ps` and wait for `postgres`, `dashboard`, and `worker` to show healthy or running.
2. Sign in using `BOOTSTRAP_EMAIL` and `BOOTSTRAP_PASSWORD` from your environment file.
3. On **Overview**, choose **Connect Portainer**.
4. Enter a private, container-reachable Portainer URL and a scoped access token. Dashbored automatically selects the first environment visible to that token; set an ID only when you need a different environment.
5. Open **Launcher** as soon as the connection is saved. The first inventory import runs immediately; the worker keeps it current every five minutes by default.

Dashbored does not seed placeholder service cards or example URLs. Your first cards come from real Portainer inventory. Add Generic Launcher entries later only for applications you want to open directly.

## Verify

```bash
docker compose ps
docker compose logs --tail=100 dashboard worker
```

PostgreSQL should remain private to the Compose network. Only the dashboard port should be published.

Continue with [installation and deployment](/docs/install) for a LAN setup or [Portainer](/docs/portainer) to connect your first provider.
